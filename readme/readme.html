Visual C++ for Linux Development

Linux Development 向けのプロジェクトの設定

このワークロードを使用して、Linux サーバー、デスクトップ、デバイス用の C++ コードを作成できます。VS からこれらのマシンへの接続を管理できます。VS では、ソースを自動的にコピーしてリモートでビルドし、デバッガーでアプリケーションを起動できます。弊社のプロジェクト システムでは、ARM を含む特定のアーキテクチャをターゲットにすることができます。

Linux への接続中

前提条件

現在サポートされているのは、Linux ターゲット マシンでのリモートでのビルドだけです。特定の Linux ディストリビューションで制限されてはいませんが、いくつかのツールの有無に依存しています。具体的には、openssh-server、g++、gdb、gdbserver が必要です。インストールするには、任意のパッケージ マネージャーを使用してください (例: Debian ベースのシステムの場合、sudo apt-get install openssh-server g++ gdb gdbserver)

最初の接続

初めて Linux マシンをターゲットとしたときには、接続情報の入力が求められます。これは、プロジェクトをビルドすることによってトリガーされます。

接続の追加と削除

新しい接続を追加するには、[ツール] > [オプション] に移動し、接続を検索してください。[接続マネージャー] は [クロス プラットフォーム] の下にあります。ここから接続を追加したり削除したりできます。

プロジェクトで使用する接続を変更するには、プロジェクトのプロパティの全般設定に移動し、[リモート ビルド マシン] オプションを更新します。

プロジェクトのプロパティ

C++ コンパイルを制御するのに必要なすべてのオプションは、プロジェクトのプロパティ ページで公開されています。Linux での動作について、いくつかの点を取り上げます。まず全般設定で、リモート ルートが既定で ~/projects/ に設定されており、リモート プロジェクト ディレクトリがその場所のプロジェクト名と一致するように設定されていることが分かります。

プロジェクトの全般設定を見ると、出力と中間ディレクトリがどのように構成されたかを確認できます。また、このプロジェクトがアプリケーションとして構成されていたため、実行可能ファイルが bin/x64/Debug/ に ConsoleApplication1.out として配置されていることがわかります。構成の種類に応じて、静的および動的ライブラリもサポートされることをご確認ください。

[リンカー] > [入力] プロパティ ページに他のライブラリ依存関係を追加します。

追加の起動前コマンドをデバッガーに渡すことにより、グラフィカル アプリの起動などの操作をリモート Linux マシン上で行うことができます。

ビルド後イベントを送信してリモート動作を制御することもできます。この例では、実行可能ファイルをスーパー ユーザーとして実行することなく GPIO ピンをエクスポートして使用します。
